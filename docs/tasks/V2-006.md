V2-006 Project Cleanup & Engineering Review

Background

Version 2 has completed its major architectural refactoring.

The project is now configuration-driven and modularized.

Before starting Version 3, perform one final engineering review to ensure the project is clean, consistent, and maintainable.

⸻

Goal

Complete the final cleanup of Version 2.

This task introduces no new features.

Only improve engineering quality.

⸻

Scope

1. Import Review

Review all Python modules.

* Remove unused imports.
* Remove duplicate imports.
* Keep import order consistent.

⸻

2. Helper Function Review

Review helper functions remaining in scripts/sync.py.

Examples:

* make_slug()
* build_article_filename()
* get_article_image_filename()
* get_article_image_path()
* get_first_image()

Determine whether they should remain in sync.py or become shared utilities in a future version.

No extraction is required in Version 2.

⸻

3. Naming Review

Review naming consistency.

Examples:

* generate_html_files()
* generate_label_pages()
* fetch_feed()
* download_images()

Record any naming improvements for Version 3.

Do not rename functions in Version 2 unless necessary.

⸻

4. Dead Code Review

Check for:

* Duplicate functions
* Unused helper functions
* Unused variables
* Unreachable code
* Old compatibility code

Remove only code confirmed to be unnecessary.

⸻

5. Documentation Review

Verify consistency between:

* architecture.md
* roadmap.md
* development.md
* configuration.md
* changelog.md
* reviews/

Update documentation if needed.

⸻

6. Architecture Review

Verify the current project structure.

Current expected architecture:

Configuration

↓

Data Source

↓

Rendering

↓

Runtime

↓

Presentation

Confirm module responsibilities remain clear.

⸻

Validation

* Python compiles successfully.
* Website generation succeeds.
* Generated output remains unchanged.
* Manual website testing passes.

⸻

Acceptance Criteria

* No functional changes.
* No UI changes.
* Cleaner project structure.
* Documentation synchronized.
* Version 2 ready for long-term maintenance.

⸻

Expected Result

Version 2 reaches a stable engineering baseline.

Version 3 can begin without additional architectural refactoring.
