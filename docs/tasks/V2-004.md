# V2-004 Preference System

## Status

📋 Planned

---

## Priority

High

---

## Milestone

Milestone B — Runtime Architecture

---

## Goal

Introduce a unified Preference System.

The frontend should no longer access localStorage directly.

All user preferences should be managed through a single Preference API.

---

## Background

Version 1 stores user preferences directly in localStorage.

Examples:

- viewMode
- sortOrder
- readingHistory

As more features are added, this approach becomes difficult to maintain.

Version 2 introduces a Preference layer between UI and localStorage.

---

---

## Scope

This version includes:

- View Preference
- Sort Preference

Future versions will reuse the same system for:

- Theme
- Font Size
- Layout
- Language
- Timeline
- Reading Mode

---

## Design Principles

UI never reads localStorage directly.

UI never writes localStorage directly.

Preference API is the only storage interface.

Preference overrides Configuration.

Configuration provides defaults only.

---

## API

Planned interface:

Preference.getView()

Preference.setView()

Preference.getSort()

Preference.setSort()

Future:

Preference.getTheme()

Preference.setTheme()
