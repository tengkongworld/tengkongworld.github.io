V2-003 Collection Default Behavior

Status

🚧 In Progress

⸻

Priority

High

⸻

Milestone

Milestone A — Configuration Driven

⸻

Goal

Move collection default behavior from hard-coded JavaScript into archive configuration.

Each Collection defines its own default presentation, while users can permanently override those defaults through local preferences.

⸻

Background

Version 1 stores page behavior directly inside JavaScript.

For example:

* Default view mode
* Default sort order

These values are hard-coded, making every Collection behave the same way.

Version 2 introduces a configuration-driven architecture.

The generator (sync.py) becomes responsible for reading configuration and writing the appropriate metadata into generated HTML.

The frontend JavaScript no longer needs to understand Collection configuration.

⸻

Design Principles

This task follows the Version 2 architecture:

Configuration

↓

sync.py

↓

Generated HTML

↓

JavaScript

↓

User Preference

Configuration defines the default behavior.

JavaScript handles only runtime interaction.

User preference always overrides configuration.

⸻

Scope

Configuration

Use values defined in:

config/archive.config.json

Example:

* default_view
* default_sort

⸻

Generator

sync.py should:

* Read Collection configuration.
* Determine the current Collection defaults.
* Generate HTML attributes for frontend use.

Example:

* data-default-view
* data-default-sort

⸻

Frontend

JavaScript should:

Read the generated HTML attributes.

If no saved preference exists:

Use the configured defaults.

If a saved preference exists:

Use the user’s preference instead.

When the user changes view or sort:

Save the new preference into localStorage.

⸻

Not Included

This task does NOT include:

* Homepage generation
* Reading history
* Theme system
* Timeline
* Search
* Collection statistics
* Layout redesign

These belong to future Version 2 tasks.

⸻

Acceptance Criteria

The implementation is considered complete when:

✓ Alien Collection opens in Gallery mode on first visit.

✓ Difficult Sutra Collection opens in List mode on first visit.

✓ Alien Collection default sort is Desc.

✓ Difficult Sutra Collection default sort is Asc.

✓ After changing view mode, refreshing the page restores the user’s preference.

✓ After clearing localStorage, the page returns to Collection defaults.

✓ JavaScript no longer contains hard-coded Collection defaults.
