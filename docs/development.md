Refactoring Rule

When performing refactoring:

1. Only one module may be extracted at a time.
2. No new features may be added during refactoring.
3. Behavior must remain identical.
4. Every extraction must pass all existing tests.
5. Commit after every successful extraction.

Every milestone must satisfy:

✓ Documentation updated

✓ Roadmap updated

✓ Architecture updated

✓ Tasks completed

✓ Validation passed

✓ Commit completed

Post-Refactoring Checklist

□ Compile

□ Run

□ Compare Output

□ Scan duplicate functions

□ Scan unused imports

□ Review module boundaries

□ Commit
